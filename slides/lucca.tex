\begin{frame}
	\frametitle{Estado da Arte}
	\framesubtitle{Estado da Arte > Plataforma \jason}

	A plataforma \jason \cite{bordini-jason} é um sistema multi-agente que
	permite definir:

	\begin{itemize}
		\item zero ou um ambiente, no qual é possível definir o mundo virtual
              que os agentes atuarão permitindo modificar percepções, definir
              ações possíveis e etc.
		\item zero ou mais agentes, no qual os agentes representam a mente por
              de trás dos atores virtuais no mundo criado.
	\end{itemize}
\end{frame}

\begin{frame}[fragile]
	\frametitle{Estado da Arte}
	\framesubtitle{Estado da Arte > Plataforma \jason}

	\begin{columns}
	\column{0.4\textwidth}
		Essa plataforma opera com o seguinte ciclo:
		\begin{enumerate}
			\item Recebimento das percepções;
			\item Raciocínio baseando-se em regras lógicas;
			\item Envio de ordens para os atuadores;
		\end{enumerate}
	\column{0.6\textwidth}
		\begin{center}
		\begin{lstlisting}
		!safety.
		!wandering.

		+!safety
		     : .findall(E, enemy(E), LE)
		     & .length(LE, LEN) & LEN >= 2
		    <- flee; !!safety.
		+!safety <- !!safety.

		+!wandering[source(self)]
		    <- forward; !!wandering.
		-!wandering[source(self)]
		    <- left; !!wandering.

		+enemy(E) <- hit(E).
		\end{lstlisting}
		\end{center}
	\end{columns}
\end{frame}

\begin{frame}
	\frametitle{Estado da Arte}
	\framesubtitle{Estado da Arte > Plataforma \jason}

	\begin{itemize}
		\item Incorporação de ontologias à plataforma foi definida em
\cite{moreira2006agent} e implementada em \cite{KlaBor09};
		\item A operação com a ontologia feita nesse não se restringe somente
em operar indivíduos permitindo acrescentar conceitos e a realização de
consultas durante a simulação;
		\item A relevância para mandar a informação para a ontologia é via
anotação e há como definir o projeto via ontologia.
		\item A implementação realizada é mais simples porque preocupa-se
somente com indivíduos e a relevância é automática baseando-se nos axiomas
existentes
	\end{itemize}

					informa o agente estádio;
				\item o ambiente é probabilístico e a cada turno um time tem a
chance de 0.83\% de fazer gol;
				\item emoções que podem acontecer são do ramo de
responsabilidade e consequência;
			\end{itemize}
	\end{columns}
\end{frame}

\begin{frame}
	\frametitle{Validação}
	\begin{columns}
		\column{0.5\textwidth}
			A validação do \emph{framework} desenvolvido foi feita com a construção de exemplos de utilização:
			\begin{itemize}
				\item testes interativos;
				\item testes não-interativos;
				\item simulador de jogo de futebol;
				\item \alert{simulador de uma casa};
			\end{itemize}
		\column{0.5\textwidth}
			\begin{itemize}
			\item 4 agentes;
			\item cada agente tem uma rotina a ser seguida diferente dos demais;
			\item a rotina pode ser ignorada por motivos emocionais; \\
                  por exemplo, cansaço que faz o agente ficar em sofrimento;
			\item as emoções presentes aqui são as mais básicas (amor e ódio);
			\end{itemize}
	\end{columns}
\end{frame}

\section{Conclusão}
% Conclusão
\begin{frame}
	\frametitle{Conclusão}
	\begin{itemize}
		\item As ontologias desenvolvidas exigem pouco conhecimento para serem usadas adequadamente
		\item Melhoras nelas podem incluir:
		\begin{itemize}
			\item Tornar a probabilidade um número inteiro;
			\item Incluir a empatia como um número e não como simplesmente a
distinção de ser a mesma pessoa ou a outra;
		\end{itemize}
		\item O sistema utilizando preferência consegue descobrir
emoções do ramo de aspectos de objetos
	\end{itemize}
\end{frame}

\begin{frame}
	\frametitle{Conclusão}
	\begin{itemize}
		\item O ``framework'' depende:
		\begin{itemize}
			\item de os limites de todas as emoções serem configuradas para o
sistema ser ativado
			\item de o usuário definir como as percepções afetam o agente
			\item de o ambiente informar uma percepção ``step'' para indicar
que é um novo ciclo de simulação
		\end{itemize}
		\item Os exemplos desenvolvidos não possui um decaimento automático
	\end{itemize}
\end{frame}
